The UUES codebase is split across a couple dozen namespaces under Uues. The Core layer has Ini for parsing config files, Crypto for AES, RSA, HMAC, and SHA operations, Net for sockets and HTTP and named pipes, Proto for protocol buffers serialization, Hook as a thin MinHook wrapper, Log for the logging subsystem, Platform for OS abstraction, Memory for low-level memory work, Config for emulator settings, Storage for save files and achievements and DLC, Threading for the thread pool and synchronization primitives, and Common for shared utility types. On top of that sit the emulator namespaces: EmuR1 handles Uplay R1, EmuR2 handles Uplay R2, EmuUPC handles Ubisoft Connect with its Demux protocol, EmuOrbit handles the old Orbit DRM. Shared has the DLL infrastructure that all the loaders rely on. Tools has utility programs and Tests has the unit and integration tests. Build instructions live under Docs/Building for each platform.
